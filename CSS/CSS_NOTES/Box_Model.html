What is the CSS Box Model?

Every HTML element is essentially a rectangular box. The CSS Box Model describes how these boxes are rendered and how their dimensions are calculated.

Key Components:

Content
The actual content (text, images, etc.)
Padding
Space between content and border
Border
Edge around the padding
Margin
Space outside the border (between elements)
/* Box Model Components */ .element { width: 200px; /* Content width */ padding: 20px; /* Inner spacing */ border: 5px solid #000; /* Border */ margin: 30px; /* Outer spacing */ }

Layer Breakdown:

  • Content: The innermost area where your actual content lives
  • Padding: Transparent space inside the border, part of the element's background
  • Border: The edge around the padding, can have different styles and colors
  • Margin: Transparent space outside the border, creates space between elements

Width and Height Calculations

Understanding how total element dimensions are calculated is crucial for layout.

Standard Box Model (content-box)

By default, when you set width or height, you're only setting the content area size.

Total Width = width + padding-left + padding-right + border-left + border-right + margin-left + margin-right
Total Height = height + padding-top + padding-bottom + border-top + border-bottom + margin-top + margin-bottom

Example:

.element { width: 200px; padding: 20px; border: 5px solid black; margin: 30px; } /* Total width = 200 + 20 + 20 + 5 + 5 + 30 + 30 = 310px */ /* Total height = (content height) + 20 + 20 + 5 + 5 + 30 + 30 */

Box-Sizing Property

The box-sizing property changes how width and height are calculated.

content-box (Default)

Content Area
width: 150px
padding: 20px
border: 5px
box-sizing: content-box;

Width/height applies only to content area. Padding and border are added to the total size.

border-box

Content Area
width: 150px
padding: 20px
border: 5px
box-sizing: border-box;

Width/height includes content, padding, and border. Padding and border are subtracted from content area.

Why Use border-box?

It makes sizing more intuitive and predictable. Many developers apply it globally:

*, *::before, *::after { box-sizing: border-box; }

Margin and Padding

Understanding the differences and proper usage of margin and padding.

Margin vs Padding:

Margin

  • Space outside the border
  • Transparent (doesn't get background)
  • Can have negative values
  • Collapses with adjacent margins
  • Used for spacing between elements

Padding

  • Space inside the border
  • Gets the element's background
  • Cannot be negative
  • No collapsing behavior
  • Used for internal spacing

Shorthand Properties:

/* All sides */ margin: 10px; padding: 15px; /* Vertical | Horizontal */ margin: 10px 20px; padding: 15px 25px; /* Top | Horizontal | Bottom */ margin: 10px 20px 30px; padding: 15px 25px 35px; /* Top | Right | Bottom | Left */ margin: 10px 20px 30px 40px; padding: 15px 25px 35px 45px;

Box Model Cheat Sheet

Box Model Properties

/* Dimensions */ width: 200px; height: 150px; min-width: 100px; max-width: 500px; min-height: 100px; max-height: 300px; /* Spacing */ padding: 20px; margin: 30px; /* Border */ border: 2px solid #000; border-radius: 10px;

Box-Sizing Values

/* Default behavior */ box-sizing: content-box; /* Width includes padding & border */ box-sizing: border-box; /* Global reset (recommended) */ *, *::before, *::after { box-sizing: border-box; } /* Shorthand for spacing */ margin: 10px 20px 30px 40px; padding: 15px 25px 35px 45px;
