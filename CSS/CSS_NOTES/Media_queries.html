What are Media Queries?

Media Queries are CSS techniques that apply different styles based on device characteristics like screen width, height, orientation, and resolution. They are the foundation of responsive web design.

Key Benefits:

Device Agnostic
One codebase works across all devices
Future-Proof
Adapts to new screen sizes automatically
User Experience
Optimized layouts for every viewport
Performance
Load appropriate assets for each device

Media Query Syntax

Media Queries use a simple syntax to target specific conditions.

Basic Structure

/* Basic media query */ @media (condition) { /* CSS rules */ } /* Common example */ @media (max-width: 768px) { .container { padding: 10px; } }

Media Query Components

Media Type
screen, print, speech
Media Feature
width, height, orientation
Logical Operators
and, or, not, only

Common Breakpoints

Breakpoints are screen widths where your layout changes. Here are industry-standard breakpoints:

📱 Mobile (Small Screens)

@media (max-width: 767px) { /* Mobile styles */ }

Phones and small devices

📱→💻 Tablet (Medium Screens)

@media (min-width: 768px) and (max-width: 1023px) { /* Tablet styles */ }

Tablets and large phones

💻 Desktop (Large Screens)

@media (min-width: 1024px) { /* Desktop styles */ }

Desktops and laptops

Mobile-First Approach

Mobile-first design starts with mobile styles and progressively enhances for larger screens using min-width queries.

Mobile-First Example

This layout adapts to screen size!
/* Base styles (mobile) */ .container { background: #e74c3c; padding: 20px; text-align: center; } /* Tablet and up */ @media (min-width: 768px) { .container { background: #3498db; padding: 30px; } } /* Desktop and up */ @media (min-width: 1024px) { .container { background: #2ecc71; padding: 40px; font-size: 1.2rem; } }
Why Mobile-First?
• Better performance on mobile devices
• Forces content prioritization
• Easier to scale up than scale down
• Future-proof for new devices

Desktop-First Approach

Desktop-first design starts with desktop styles and scales down for smaller screens using max-width queries.

Desktop-First Example

/* Base styles (desktop) */ .grid { display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 15px; } /* Tablet and down */ @media (max-width: 1023px) { .grid { grid-template-columns: 1fr 1fr; } } /* Mobile and down */ @media (max-width: 767px) { .grid { grid-template-columns: 1fr; } }
When to Use Desktop-First?
• Legacy projects
• Complex desktop applications
• When mobile isn't a priority
• Converting existing desktop sites

Advanced Media Features

Media Queries can target more than just screen width!

Orientation

@media (orientation: landscape) { /* Landscape styles */ } @media (orientation: portrait) { /* Portrait styles */ }

Resolution

@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) { /* High DPI/Retina styles */ }

Aspect Ratio

@media (aspect-ratio: 16/9) { /* 16:9 aspect ratio */ } @media (min-aspect-ratio: 4/3) { /* Wider than 4:3 */ }

Hover Capability

@media (hover: hover) { /* Devices with hover capability */ button:hover { background: #3498db; } } @media (hover: none) { /* Touch devices */ button { padding: 15px; } }

Media Queries Best Practices

✅ Use Mobile-First When Possible
Start small and enhance up for better performance
✅ Use Relative Units
Use em, rem, % instead of px for better scalability
✅ Content-Based Breakpoints
Break when content looks bad, not at arbitrary widths
✅ Test on Real Devices
Browser dev tools aren't always accurate
