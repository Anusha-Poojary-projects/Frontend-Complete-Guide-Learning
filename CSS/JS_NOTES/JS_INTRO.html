What is JavaScript?

JavaScript is a high-level, dynamic, and interpreted programming language that conforms to the ECMAScript specification. It's one of the core technologies of the World Wide Web, alongside HTML and CSS.

Client-Side Scripting

Executes in web browsers to create interactive web pages

Server-Side Capabilities

With Node.js, JavaScript runs on servers

Multi-Paradigm

Supports object-oriented, imperative, and functional programming

Dynamic Typing

Variables can hold any data type and change types during execution

The Birth of JavaScript

JavaScript was created in 1995 by Brendan Eich while working at Netscape Communications Corporation.

May 1995

Brendan Eich hired by Netscape with the assignment to put Scheme in the browser

May-June 1995

Eich develops the first version of JavaScript in just 10 days

September 1995

Netscape Navigator 2.0B3 released with LiveScript (JavaScript's original name)

December 1995

Renamed to JavaScript to capitalize on Java's popularity (despite having no relation to Java)

Why the name "JavaScript"?

Despite the name, JavaScript has no relation to Java. The name was chosen for marketing purposes during the Java hype of the mid-90s. Netscape and Sun Microsystems (creators of Java) had a partnership, and the name was meant to suggest it was a "younger brother" to Java.

Standardization: ECMAScript

As JavaScript gained popularity, Microsoft created their own implementation called JScript for Internet Explorer. This led to compatibility issues, prompting the need for standardization.

November 1996

Netscape submits JavaScript to ECMA International for standardization

June 1997

ECMAScript 1 (ES1) specification published

December 1999

ECMAScript 3 (ES3) released - became the dominant standard for over a decade

December 2009

ECMAScript 5 (ES5) released - major update that revitalized the language

ECMAScript vs JavaScript

ECMAScript is the standard, while JavaScript is the most popular implementation of that standard. Other implementations include JScript (Microsoft) and ActionScript (Adobe).

The Dark Ages & Renaissance

After ES3 in 1999, JavaScript entered a period of stagnation, followed by a renaissance that transformed it into a modern language.

2000-2008

The "Dark Ages" - minimal language development, focus on libraries like jQuery to handle browser inconsistencies

2009

ECMAScript 5 released with JSON support, strict mode, and array methods

2011

jQuery reaches peak popularity, used by over 50% of websites

2015

ECMAScript 2015 (ES6) released - the most significant update with classes, modules, arrow functions, and more

"ES6 is such a big update that it's basically a new language layered on top of the old one."
- Kyle Simpson

Modern JavaScript

Since ES2015, ECMAScript has adopted a yearly release cycle, bringing continuous improvements to the language.

ES2016 (ES7)

Array.prototype.includes(), exponentiation operator (**)

ES2017 (ES8)

Async/await, Object.values(), Object.entries()

ES2018 (ES9)

Rest/Spread properties, asynchronous iteration

ES2019 (ES10)

Array.prototype.flat(), Object.fromEntries()

ES2020 (ES11)

Optional chaining, nullish coalescing, BigInt

ES2021+ (ES12+)

Logical assignment operators, numeric separators, top-level await

Full-Stack JavaScript

With Node.js (2009), JavaScript became a full-stack language, allowing developers to use the same language on both client and server sides.
