What are Loops?

Loops are control structures that allow you to execute a block of code repeatedly until a specified condition is met.

Why Use Loops?

  • Automation: Perform repetitive tasks without writing duplicate code
  • Efficiency: Process large datasets with minimal code
  • Flexibility: Handle dynamic data sizes
  • Maintenance: Easier to update logic in one place
  • Performance: Optimized for iteration operations

The for Loop

The for loop is the most common loop type, providing a concise way to write loops with initialization, condition, and final expression.

Basic Syntax

for (let i = 0; i < 5; i++) { console.log(i); } // Output: 0, 1, 2, 3, 4

Counting Backwards

for (let i = 10; i >= 0; i--) { console.log(i); } // Output: 10, 9, 8, ..., 0

Custom Step

for (let i = 0; i <= 20; i += 2) { console.log(i); } // Output: 0, 2, 4, ..., 20

for Loop Components

for ( [initialization]; // Executed once before loop starts [condition]; // Checked before each iteration [final-expression] // Executed after each iteration ) { // Loop body }

while and do...while Loops

These loops continue executing as long as a specified condition evaluates to true.

while Loop

let count = 0; while (count < 3) { console.log(count); count++; } // Output: 0, 1, 2

do...while Loop

let count = 0; do { console.log(count); count++; } while (count < 3); // Output: 0, 1, 2

Key Differences

  • while: Condition checked before each iteration (may not run at all)
  • do...while: Condition checked after each iteration (runs at least once)

When to Use Each

  • Use while when you don't know how many iterations you need
  • Use do...while when you need to execute at least once
  • Example: Menu systems, user input validation

for...of and for...in Loops

These specialized loops are designed for iterating over iterable objects and object properties.

for...of Loop

Iterates over iterable objects (arrays, strings, maps, etc.)

const fruits = ["apple", "banana", "orange"]; for (const fruit of fruits) { console.log(fruit); } // Output: apple, banana, orange

for...in Loop

Iterates over enumerable properties of an object

const person = { name: "John", age: 30 }; for (const key in person) { console.log(key, person[key]); } // Output: name John, age 30

Key Differences

  • for...of: Gets values directly, works with iterables
  • for...in: Gets property names/keys, works with objects
  • Arrays: Use for...of for values, for...in for indices (but avoid for...in with arrays)
// Array iteration - prefer for...of const numbers = [1, 2, 3]; for (const num of numbers) { // Good console.log(num); } for (const index in numbers) { // Avoid with arrays console.log(numbers[index]); }

Array Iteration Methods

JavaScript arrays provide powerful built-in methods for iteration that are often more expressive than traditional loops.

forEach()
Execute function for each element
map()
Create new array by transforming elements
filter()
Create new array with elements that pass test
reduce()
Reduce array to single value
find()
Return first element that matches condition
some()/every()
Check if some/every element matches condition

Examples

const numbers = [1, 2, 3, 4, 5]; // forEach - side effects numbers.forEach(num => console.log(num)); // map - transformation const doubled = numbers.map(num => num * 2); // filter - selection const evens = numbers.filter(num => num % 2 === 0); // reduce - accumulation const sum = numbers.reduce((acc, num) => acc + num, 0);

Loop Control Statements

JavaScript provides statements to control the flow of loops, allowing you to skip iterations or exit early.

break Statement

Exits the loop immediately

for (let i = 0; i < 10; i++) { if (i === 5) break; console.log(i); } // Output: 0, 1, 2, 3, 4

continue Statement

Skips current iteration and continues with next

for (let i = 0; i < 5; i++) { if (i === 2) continue; console.log(i); } // Output: 0, 1, 3, 4

Nested Loops

break and continue affect only the innermost loop by default

outer: for (let i = 0; i < 3; i++) { for (let j = 0; j < 3; j++) { if (i === 1 && j === 1) break outer; console.log(i, j); } } // Output: 0 0, 0 1, 0 2, 1 0

Best Practices

  • Use break to exit early when condition is met
  • Use continue to skip unwanted iterations
  • Avoid deeply nested loops when possible
  • Consider labeled statements for complex nested loop control

Loop Comparison & Best Practices

Choosing the right loop type and following best practices leads to cleaner, more efficient code.

When to Use Which Loop

Loop Type Best Use Case Performance
for Known number of iterations, arrays with indices Fastest
while Unknown iterations, condition-based looping Fast
do...while Must execute at least once Fast
for...of Iterating over iterable objects (arrays, strings) Good
Array methods Functional programming, transformations, filtering Good (with optimizations)

Avoid Infinite Loops

Always ensure loop conditions will eventually become false

// Bad - infinite loop while (true) { ... } // Good - with exit condition let attempts = 0; while (attempts < maxAttempts) { ... }

Prefer Array Methods

Use functional methods when possible for readability

// Instead of for loop const doubled = []; for (let i = 0; i < numbers.length; i++) { doubled.push(numbers[i] * 2); } // Use map const doubled = numbers.map(n => n * 2);

Cache Length

Cache array length in for loops for better performance

// Good for (let i = 0, len = arr.length; i < len; i++) { ... } // Avoid (in older engines) for (let i = 0; i < arr.length; i++) { ... }

Meaningful Variables

Use descriptive variable names in loops

// Good for (const user of users) { ... } for (const product of products) { ... } // Avoid for (const x of y) { ... }

JavaScript Loops & Iteration: Key Takeaways

Loop Types

  • for: Traditional loop with counter
  • while/do...while: Condition-based looping
  • for...of: Iterate over iterable values
  • for...in: Iterate over object properties
  • Array methods: Functional iteration (map, filter, etc.)

Best Practices

  • Choose the right loop for the job
  • Prefer array methods for functional programming
  • Use meaningful variable names
  • Avoid infinite loops
  • Cache array lengths in performance-critical code
