What are Conditional Statements?

Conditional statements allow your program to make decisions and execute different code blocks based on whether a condition is true or false.

Why Use Conditionals?

  • Decision Making: Execute different code paths based on conditions
  • Input Validation: Check if user input meets requirements
  • Error Handling: Handle different error scenarios
  • Feature Toggling: Enable/disable features based on conditions
  • Game Logic: Implement game rules and win/lose conditions

The if Statement

The if statement executes a block of code if a specified condition is true.

Basic if

if (age >= 18) { console.log("You are an adult"); }

if...else

if (temperature > 30) { console.log("It's hot!"); } else { console.log("It's not hot."); }

if...else if...else

if (score >= 90) { grade = "A"; } else if (score >= 80) { grade = "B"; } else if (score >= 70) { grade = "C"; } else { grade = "F"; }

Syntax Rules

  • Conditions are enclosed in parentheses ()
  • Code blocks are enclosed in curly braces {}
  • Curly braces can be omitted for single statements (but not recommended)
  • Conditions are evaluated as boolean values

Logical Operators in Conditions

Logical operators allow you to combine multiple conditions to create more complex decision logic.

Logical AND (&&)

Returns true only if both conditions are true

// User must be logged in AND have admin privileges if (isLoggedIn && isAdmin) { showAdminPanel(); } // Short-circuit evaluation let name = user && user.name; // Safe property access

Logical OR (||)

Returns true if at least one condition is true

// User can access if they are admin OR have permission if (isAdmin || hasPermission) { grantAccess(); } // Default values let displayName = name || "Guest";

Logical NOT (!)

Negates a condition (true becomes false, false becomes true)

// Check if user is NOT logged in if (!isLoggedIn) { redirectToLogin(); } // Double negation converts to boolean let isTruthy = !!value;

The switch Statement

The switch statement evaluates an expression and executes the code block that matches the expression's value.

switch (expression)
case value1:
// code block
break;
case value2:
// code block
break;
default:
// default code

Example: Day of Week

switch (day) { case 1: console.log("Monday"); break; case 2: console.log("Tuesday"); break; case 3: console.log("Wednesday"); break; // ... more cases default: console.log("Invalid day"); }

Important Notes

  • break statements: Prevent fall-through to the next case
  • default case: Executes when no cases match
  • Strict comparison: Uses === for comparison
  • Fall-through: Multiple cases can share the same code block

When to Use Ternary

  • Simple assignments based on conditions
  • Returning values from functions
  • Setting default values
  • Conditional rendering in JSX (React)

When to Avoid

  • Nested ternary operators (hard to read)
  • Complex logic with multiple statements
  • When side effects are involved

Conditional Statements Best Practices

Following best practices leads to cleaner, more maintainable, and bug-free code.

Early Returns

Return early from functions to reduce nesting

// Good function processUser(user) { if (!user) return; if (!user.isActive) return; // Process user... }

Avoid Deep Nesting

Use guard clauses and early returns

// Avoid if (user) { if (user.isActive) { if (user.hasPermission) { // ... } } }

Meaningful Conditions

Use descriptive variable names

// Good const isUserEligible = user.age >= 18 && user.isVerified; if (isUserEligible) { ... } // Avoid if (user.age >= 18 && user.isVerified) { ... }

Consistent Formatting

Always use curly braces

// Good if (condition) { doSomething(); } // Avoid if (condition) doSomething();

Performance Considerations

  • Place most likely conditions first in if...else chains
  • Use switch for many equality comparisons
  • Avoid expensive operations in conditions
  • Cache computed values when used multiple times

JavaScript Conditional Statements: Key Takeaways

Conditional Types

  • if/else: For complex conditions and multiple statements
  • switch: For multiple equality comparisons
  • Ternary: For simple assignments and returns
  • Logical operators: For combining conditions

Core Concepts

  • Understanding truthy/falsy values
  • Short-circuit evaluation with && and ||
  • Strict comparison in switch statements
  • Early returns to reduce complexity
