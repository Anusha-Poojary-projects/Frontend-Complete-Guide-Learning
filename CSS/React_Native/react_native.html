Styling in React Native

Styling uses JavaScript objects instead of CSS, with a similar but not identical syntax.

Basic Styling

import { StyleSheet } from 'react-native'; const styles = StyleSheet.create({ container: { flex: 1, backgroundColor: '#fff', alignItems: 'center', justifyContent: 'center', padding: 20, }, title: { fontSize: 24, fontWeight: 'bold', color: '#2193b0', marginBottom: 10, }, }); // Usage Styled Component

Key Differences from CSS

  • No inheritance - styles must be applied to each component
  • Flexbox is the default layout system (with some differences)
  • No shorthand properties (must specify each property separately)
  • Dimensions are unitless (values are in density-independent pixels)

Styling Best Practices

  • Use StyleSheet.create for performance optimization
  • Organize styles in separate files for complex components
  • Use consistent naming conventions
  • Consider using styled-components or other CSS-in-JS libraries

Interview Questions - Beginner Level

BEGINNER
What is React Native and how does it differ from React?
React Native is a framework for building native mobile applications using React and JavaScript. While React is used for building web applications with HTML/CSS, React Native uses native components instead of web elements. React Native compiles to actual native UI components, providing a truly native look and feel, whereas React renders to the DOM.
Explain the difference between View and Text components.
View is a container component that supports layout with flexbox, style, touch handling, and accessibility controls. It's equivalent to a div in web development. Text is used for displaying text content and supports nesting, styling, and touch handling. Unlike web, all text must be wrapped in a Text component in React Native - you cannot render text directly inside a View.
What is the purpose of StyleSheet.create()?
StyleSheet.create() is used to create style objects that are optimized for performance. It validates the styles during creation and assigns them unique IDs, which allows React Native to send the style information efficiently to the native side. While you can use plain JavaScript objects for styles, StyleSheet.create() is the recommended approach for better performance and code organization.

Interview Questions - Intermediate Level

INTERMEDIATE
How does navigation work in React Native? Explain the differences between Stack, Tab, and Drawer navigators.
React Navigation is the standard library for navigation in React Native. Stack Navigator manages screens in a stack (like web pages), where you push and pop screens. Tab Navigator provides bottom or top tabs for switching between primary sections of an app. Drawer Navigator implements a slide-in menu from the side, typically used for less frequently accessed screens. These navigators can be nested to create complex navigation structures.
What are the differences between useState and useReducer? When would you use each?
useState is simpler and used for managing single primitive values or simple objects. useReducer is better for complex state logic with multiple sub-values or when the next state depends on the previous one. useReducer is also preferred when you have related state updates that should happen together, as it centralizes state transition logic and makes testing easier. For simple state, useState is sufficient; for complex state, useReducer provides better organization.
How do you handle API calls in React Native? What are the best practices?
API calls are typically handled using fetch or axios within useEffect hooks. Best practices include: using async/await for cleaner code, implementing proper error handling, showing loading states to users, canceling requests on component unmount to prevent memory leaks, using environment variables for API endpoints, and considering libraries like React Query for advanced data fetching with caching and background updates.

Interview Questions - Advanced Level

ADVANCED
Explain how the React Native Bridge works and its limitations.
The React Native Bridge is a communication layer between JavaScript and native code. It's asynchronous and serialized, meaning data is passed as JSON messages. This creates a performance bottleneck for high-frequency communication. The bridge is also single-threaded, so heavy JavaScript operations can block UI updates. Newer architectures like Fabric (part of the New Architecture) aim to eliminate the bridge by using a synchronous, direct communication channel with C++ shared memory.
How would you optimize a React Native app that's experiencing performance issues?
Performance optimization strategies include: using PureComponent or React.memo to prevent unnecessary re-renders, optimizing FlatList with proper props (getItemLayout, initialNumToRender), using native driver for animations, implementing code splitting, reducing bundle size by removing unused dependencies, using Hermes engine, implementing proper memory management (cleaning up timers/listeners), and using performance monitoring tools like Flipper to identify bottlenecks.
Describe the React Native New Architecture and its benefits.
The New Architecture replaces the legacy bridge with synchronous, direct communication between JavaScript and native code using Fabric (UI renderer) and TurboModules (native modules). Benefits include: improved performance with synchronous calls, better memory efficiency, support for concurrent rendering, reduced startup time, and the ability to share code between platforms using C++. This architecture enables features like synchronous APIs and better integration with React 18 features.

Common Interview Questions

How do you handle different screen sizes and orientations in React Native?
Use responsive design techniques: Flexbox for layout, percentage-based dimensions, useWindowDimensions hook for dynamic sizing, and platform-specific styles when needed. For orientation changes, listen to dimension changes with useWindowDimensions or Dimensions API. Libraries like react-native-responsive-screen can help with consistent sizing across devices. Always test on multiple device sizes during development.
What strategies do you use for state management in large React Native applications?
For large applications, I prefer a layered approach: Context API for simple global state (like auth), Redux Toolkit or Zustand for complex application state, and React Query for server state. I avoid prop drilling by creating domain-specific contexts. I also follow the principle of colocating state as close to where it's used as possible, only lifting state up when necessary for sharing.
How do you ensure your React Native app is accessible?
I follow platform accessibility guidelines: use semantic components with proper accessibilityRole, provide accessibilityLabel for interactive elements, ensure sufficient color contrast, create adequate touch targets (minimum 44x44pt), support dynamic text sizing, and test with VoiceOver (iOS) and TalkBack (Android). I also use accessibility inspectors in Xcode and Android Studio during development.

Behavioral Interview Questions

Describe a challenging bug you encountered in a React Native app and how you solved it.
In a previous project, we had a memory leak causing the app to crash after extended use. Using Flipper's memory inspector, I identified that event listeners weren't being removed on component unmount. I implemented proper cleanup in useEffect return functions and used React.memo to prevent unnecessary re-renders that were creating new listener instances. This reduced memory usage by 60% and eliminated the crashes.
How do you stay updated with React Native developments?
I follow the official React Native blog and GitHub repository, subscribe to newsletters like React Native Newsletter, participate in communities like React Native Discord and Reddit, attend conferences like Chain React, and experiment with new features in personal projects. I also regularly review the React Native changelog to understand breaking changes and new capabilities.
Tell me about a time you had to optimize a slow React Native app.
I worked on an e-commerce app with a product listing screen that was extremely slow. Analysis showed the FlatList was re-rendering all items on every scroll. I implemented getItemLayout, used React.memo for list items, moved heavy computations out of render, and implemented proper image caching. I also switched to Hermes engine and enabled Proguard for the Android build. These changes reduced scroll jank from 30fps to 60fps consistently.

System Design Questions

How would you architect a React Native app for a social media platform?
I'd use a modular architecture with feature-based organization. State management would combine Redux Toolkit for global state (user auth, notifications) and React Query for server state (posts, comments). Navigation would use nested navigators (tabs for main sections, stack for detail views). I'd implement offline support with WatermelonDB, push notifications with Firebase, and real-time updates with WebSockets. Performance would be monitored with Sentry, and CI/CD would be set up with GitHub Actions and Fastlane.
Design a real-time chat application using React Native.
The architecture would include: WebSocket connection for real-time communication, React Query for message history with background refetching, Zustand for local chat state, native notifications for background messages, and secure authentication with JWT. Messages would be stored locally with AsyncStorage for offline access and synced when online. I'd use FlatList with inverted scrolling for the chat view and implement proper typing indicators and read receipts. Security would include end-to-end encryption for sensitive messages.
How would you implement offline-first functionality in a data-intensive app?
I'd use WatermelonDB as the local database for its reactive, observable queries and efficient sync capabilities. All user actions would be queued locally and synced when connectivity is restored. Conflict resolution would follow a last-write-wins strategy with timestamps. The UI would clearly indicate offline status and sync progress. For large file attachments, I'd implement progressive upload/download with pause/resume capability. Data would be encrypted at rest using SQLCipher.
