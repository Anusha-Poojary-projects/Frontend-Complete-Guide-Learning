Prerequisites

Before creating a React app, ensure you have these installed:

Node.js & npm

Node.js (version 14.0 or newer) is required to run JavaScript outside the browser.

npm (Node Package Manager) comes bundled with Node.js and is used to install packages.

# Check your versions node --version npm --version
Code Editor

While any text editor works, these are recommended:

  • Visual Studio Code (most popular with excellent React support)
  • WebStorm (powerful IDE with built-in React tools)
  • Sublime Text (lightweight with React plugins)
Browser

Modern browsers with developer tools:

  • Google Chrome (with React Developer Tools extension)
  • Mozilla Firefox
  • Microsoft Edge

Creating a React App

The easiest way to create a new React app is using Create React App (CRA).

Using npx (Recommended)

npx is a package runner tool that comes with npm 5.2+. It allows you to run packages without installing them globally.

# Create a new React app named 'my-app' npx create-react-app my-app # Navigate to your app directory cd my-app # Start the development server npm start

Alternative Methods

  • Using npm: npm init react-app my-app
  • Using Yarn: yarn create react-app my-app
  • Vite (Modern Alternative): npm create vite@latest my-app -- --template react

What Happens During Setup?

  • Creates a project directory with all necessary files
  • Installs React, ReactDOM, and development dependencies
  • Sets up Webpack, Babel, and ESLint with sensible defaults
  • Configures development server with hot reloading

React App Folder Structure

Understanding the default folder structure is crucial for efficient development.

my-app/
├── node_modules/ # Installed dependencies
├── public/ # Static assets served directly
│ ├── favicon.ico
│ ├── index.html # Main HTML template
│ └── manifest.json # Web app manifest
├── src/ # Source code directory
│ ├── App.css # App component styles
│ ├── App.js # Main App component
│ ├── App.test.js # App component tests
│ ├── index.css # Global styles
│ ├── index.js # Entry point
│ ├── logo.svg # React logo
│ └── reportWebVitals.js # Performance monitoring
├── .gitignore # Files to ignore in Git
├── package.json # Project metadata and dependencies
├── package-lock.json # Exact dependency versions
└── README.md # Project documentation

Key Directories

  • public/: Contains static files that are served directly by the web server
  • src/: Contains all your React components, styles, and application logic
  • node_modules/: Contains all installed npm packages (should be in .gitignore)

Dependencies Overview

React apps rely on various dependencies for development and production.

Core Dependencies

  • react: The React library itself
  • react-dom: Provides DOM-specific methods

Development Dependencies

  • react-scripts: Configuration and scripts for CRA
  • web-vitals: Performance monitoring

Build Tools (Included)

  • Webpack: Module bundler
  • Babel: JavaScript compiler
  • ESLint: Code linter
  • Jest: Testing framework

Optional but Recommended

  • prop-types: Runtime type checking
  • react-router-dom: Routing
  • axios: HTTP client

Viewing Dependencies

# View all dependencies npm list --depth=0 # View production dependencies only npm list --depth=0 --production # View development dependencies only npm list --depth=0 --dev

Essential Dependencies to Install

While Create React App includes many tools, you'll often need to add these common dependencies.

React Router

For client-side routing

npm install react-router-dom
Axios

Promise-based HTTP client

npm install axios
Styled Components

CSS-in-JS styling

npm install styled-components
Redux Toolkit

State management

npm install @reduxjs/toolkit react-redux
Formik

Form handling

npm install formik
React Icons

Popular icon libraries

npm install react-icons

Installing Dependencies

  • Production dependencies: npm install package-name
  • Development dependencies: npm install --save-dev package-name
  • Global packages: npm install -g package-name (rarely needed)

Development Scripts

Create React App provides several useful scripts in package.json.

npm start

Runs the app in development mode with hot reloading.

Local: http://localhost:3000
On Your Network: http://192.168.x.x:3000

npm test

Launches the test runner in watch mode.

Runs tests related to files changed since last commit.

npm run build

Builds the app for production to the build folder.

Optimizes for performance with minification and code splitting.

npm run eject

⚠️ One-way operation! Exposes all configuration files.

Allows customization of webpack, Babel, etc. but removes CRA abstraction.

Custom Scripts

You can add your own scripts to package.json:

{ "scripts": { "start": "react-scripts start", "build": "react-scripts build", "test": "react-scripts test", "eject": "react-scripts eject", "lint": "eslint src", "format": "prettier --write src" } }

Creating a React App: Key Takeaways

Setup Process

  • Install Node.js and npm
  • Use npx create-react-app my-app
  • Run npm start to launch development server
  • Start coding in the src directory

Folder Structure

  • public/: Static assets and HTML template
  • src/: All React components and logic
  • index.js: Entry point that renders App
  • App.js: Root component of your application
