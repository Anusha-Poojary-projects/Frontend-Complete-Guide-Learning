üåü What is SQL?

SQL in Simple Terms

SQL (Structured Query Language) is how we talk to databases. Think of it like asking a librarian for books:

  • ‚Ä¢ "Show me all books by Stephen King" ‚Üí SELECT * FROM books WHERE author = 'Stephen King'
  • ‚Ä¢ "Add this new book to the catalog" ‚Üí INSERT INTO books VALUES (...)
  • ‚Ä¢ "Change the price of this book" ‚Üí UPDATE books SET price = 15.99 WHERE id = 123

‚ûï Create Tables

Creating Your First Table

Tables are like spreadsheets with rows and columns. Let's create a users table:

-- Create a users table
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE,
  age INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

üí° What These Mean:

  • ‚Ä¢ PRIMARY KEY - Unique ID for each row
  • ‚Ä¢ NOT NULL - This field can't be empty
  • ‚Ä¢ UNIQUE - No two rows can have the same value
  • ‚Ä¢ DEFAULT - Use this value if none is provided

Common Data Types

Type What it stores Example

VARCHAR(100) Text (up to 100 characters) "John Doe"
TEXT Long text "This is a long description..."

BOOLEAN True/False TRUE, FALSE

üì• Insert Data

Adding Data to Your Table

Now let's add some users to our table:

-- Add one user
INSERT INTO users (name, email, age) VALUES
('Alice Johnson', 'alice@example.com', 28);

-- Add multiple users
INSERT INTO users (name, email, age) VALUES
('Bob Smith', 'bob@example.com', 32),
('Carol Davis', 'carol@example.com', 25);

üí° Tips for INSERT:

  • ‚Ä¢ List column names to avoid confusion
  • ‚Ä¢ Add multiple rows at once for efficiency
  • ‚Ä¢ Text values need quotes, numbers don't
  • ‚Ä¢ The id and created_at will be filled automatically

‚ö†Ô∏è Watch Out For:

  • ‚Ä¢ Don't forget commas between values
  • ‚Ä¢ Match the number of columns with values
  • ‚Ä¢ Respect NOT NULL and UNIQUE constraints
  • ‚Ä¢ Use single quotes for text, not double quotes

üîç Select Data

Reading Data from Your Table

SELECT is how you view data in your database:

-- Show all users
SELECT * FROM users;

-- Show only name and email
SELECT name, email FROM users;

-- Show with custom column names
SELECT name AS 'Full Name', email AS 'Email Address' FROM users;

üí° SELECT Tips:

  • ‚Ä¢ Use * to select all columns
  • ‚Ä¢ List specific columns to improve performance
  • ‚Ä¢ Use AS to give columns friendly names
  • ‚Ä¢ Always end SQL statements with a semicolon (;)

üéØ Filter Data with WHERE

Finding Specific Data

Use WHERE to filter your results:

-- Users over 30
SELECT * FROM users WHERE age > 30;

-- Specific user by email
SELECT * FROM users WHERE email = 'alice@example.com';

-- Users between 25 and 30
SELECT * FROM users WHERE age >= 25 AND age <= 30;

-- Users whose name starts with 'A'
SELECT * FROM users WHERE name LIKE 'A%';

üí° WHERE Operators:

= (equals)
!= or <> (not equals)
> (greater than)
< (less than)
>= (greater than or equal)
<= (less than or equal)
LIKE (pattern matching)
IN (match any in list)

Pattern Matching with LIKE

WHERE name LIKE 'A%'

Names starting with A (Alice, Andrew, etc.)

WHERE name LIKE '%son'

Names ending with son (Johnson, Jackson, etc.)

WHERE name LIKE '%a%'

Names containing 'a' (Alice, Carol, etc.)

WHERE name LIKE '_a%'

Names with 'a' as second letter (Carol, James, etc.)

‚úèÔ∏è Update Data

Changing Existing Data

Use UPDATE to modify data in your table:

-- Change Alice's age
UPDATE users SET age = 29 WHERE name = 'Alice Johnson';

-- Change email and age for Bob
UPDATE users SET email = 'bob.smith@example.com', age = 33 WHERE id = 2;

-- Add 1 to everyone's age (be careful!)
UPDATE users SET age = age + 1;

‚ö†Ô∏è IMPORTANT: Always use WHERE with UPDATE!

If you forget the WHERE clause, you'll update ALL rows in the table!

UPDATE users SET age = 30; ‚Üê This changes everyone's age!

üí° UPDATE Best Practices:

  • ‚Ä¢ Always test your WHERE clause with SELECT first
  • ‚Ä¢ Update one column at a time when learning
  • ‚Ä¢ Use transactions for important updates (COMMIT/ROLLBACK)
  • ‚Ä¢ Keep backups of important data

üóëÔ∏è Delete Data

Removing Data from Your Table

Use DELETE to remove rows from your table:

-- Delete Carol's record
DELETE FROM users WHERE name = 'Carol Davis';

-- Delete users under 18
DELETE FROM users WHERE age < 18;

-- Delete all users (be very careful!)
DELETE FROM users;

‚ö†Ô∏è DANGER: DELETE is permanent!

Unlike UPDATE, DELETE removes entire rows from your table.

DELETE FROM users; ‚Üê This deletes ALL users!

Always double-check your WHERE clause before executing!

üí° DELETE Safety Tips:

  • ‚Ä¢ Test your WHERE clause with SELECT first
  • ‚Ä¢ Consider using "soft delete" (add an "is_deleted" column)
  • ‚Ä¢ Keep regular backups of your database
  • ‚Ä¢ Use transactions so you can ROLLBACK if needed

üîó Join Tables

Combining Data from Multiple Tables

JOIN lets you combine data from related tables:

-- Create a products table
CREATE TABLE products (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100),
  price DECIMAL(10,2),
  category_id INT
);

-- Create an orders table
CREATE TABLE orders (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT,
  product_id INT,
  quantity INT
);

-- Show orders with user and product names
SELECT users.name, products.name, orders.quantity
FROM orders
JOIN users ON orders.user_id = users.id
JOIN products ON orders.product_id = products.id;

üí° JOIN Types:

  • ‚Ä¢ JOIN or INNER JOIN - Only matching rows from both tables
  • ‚Ä¢ LEFT JOIN - All rows from left table, matching from right
  • ‚Ä¢ RIGHT JOIN - All rows from right table, matching from left
  • ‚Ä¢ FULL JOIN - All rows from both tables
