✏️ Update Documents — Complete Guide

Update Operation Flow

1
Identify documents to update using filter criteria
2
Specify update operations using update operators
3
Choose between updateOne() or updateMany()
4
Execute update and review results

📋 Update Operators Cheat Sheet

$set - Set field value
$unset - Remove field
$inc - Increment number
$mul - Multiply number
$rename - Rename field
$min - Set if smaller
$max - Set if larger
$currentDate - Set to current date

Example 1: updateOne() with $set

db.users.updateOne(
  { name: "Alice Johnson" },
  { $set: { age: 29, city: "Boston" } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $set operator sets the value of a field. If the field doesn't exist, it will be created.

Example 2: updateMany() with $inc

db.products.updateMany(
  { category: "Electronics" },
  { $inc: { price: 10, stock: -1 } }
})
{ "acknowledged": true, "matchedCount": 5, "modifiedCount": 5 }

💡 $inc operator increments a numeric field. Use negative values to decrement.

Example 3: Remove fields with $unset

db.users.updateOne(
  { name: "Alice Johnson" },
  { $unset: { email: "", city: "" } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $unset operator removes the specified fields from documents.

Example 4: Rename fields with $rename

db.users.updateMany(
  {},
  { $rename: { "zipcode": "postalCode" } }
})
{ "acknowledged": true, "matchedCount": 3, "modifiedCount": 3 }

💡 $rename operator changes the name of a field. The field must exist.

Example 5: Multiply values with $mul

db.products.updateMany(
  { onSale: true },
  { $mul: { price: 0.9 } }
})
{ "acknowledged": true, "matchedCount": 2, "modifiedCount": 2 }

💡 $mul operator multiplies the value of a field by the specified number.

Example 6: Set minimum/maximum with $min/$max

db.products.updateOne(
  { _id: "PROD001" },
  {
    $min: { price: 25 },
    $max: { discount: 50 }
  }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $min sets the field if the specified value is less than the current value. $max sets if greater.

Example 7: Set current date with $currentDate

db.users.updateOne(
  { name: "Alice Johnson" },
  { $currentDate: { lastUpdated: true } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $currentDate sets a field to the current date. Use { lastUpdated: { $type: "timestamp" } } for timestamp.

Example 8: Upsert (update or insert)

db.users.updateOne(
  { email: "newuser@example.com" },
  { $set: { name: "New User", age: 25 } },
  { upsert: true }
})
{ "acknowledged": true, "matchedCount": 0, "modifiedCount": 0, "upsertedId": ObjectId("64a1b2c3d4e5f6a7b8c9d0e5") }

💡 upsert: true creates a new document if no document matches the filter.

🚀 Real-World Update Use Cases

• User profile updates: Modify user information, preferences, or settings
• Inventory management: Update product stock levels after purchases
• Analytics: Increment view counts or engagement metrics
• Session management: Update last activity timestamp for user sessions
• Pricing updates: Adjust product prices based on promotions or market changes

🗑️ Delete Documents — Complete Guide

Delete Operation Flow

1
Identify documents to delete using filter criteria
2
Choose between deleteOne() or deleteMany()
3
Execute delete operation
4
Review results (deletedCount)

Example 1: deleteOne() by _id

db.users.deleteOne({ _id: ObjectId("64a1b2c3d4e5f6a7b8c9d0e1") })
{ "acknowledged": true, "deletedCount": 1 }

💡 deleteOne() removes a single document that matches the filter criteria.

Example 2: deleteMany() with conditions

db.orders.deleteMany({ status: "cancelled", createdAt: { $lt: new Date("2023-01-01") } })
{ "acknowledged": true, "deletedCount": 15 }

💡 deleteMany() removes all documents that match the filter criteria.

Example 3: Delete ALL documents

db.temp_data.deleteMany({})
{ "acknowledged": true, "deletedCount": 127 }

💡 Empty filter {} matches all documents in the collection. Use with caution!

Example 4: Delete documents with array conditions

db.users.deleteMany({ tags: { $in: ["inactive", "spam"] } })
{ "acknowledged": true, "deletedCount": 8 }

💡 Use array operators like $in to delete documents based on array content.

Example 5: Delete with nested document conditions

db.users.deleteMany({ "address.country": "Unknown" })
{ "acknowledged": true, "deletedCount": 3 }

💡 Use dot notation to delete documents based on fields in nested documents.

🚀 Real-World Delete Use Cases

• User account deletion: Remove user data upon request (GDPR compliance)
• Data cleanup: Remove old logs, temporary files, or expired sessions
• Order management: Delete cancelled orders after a retention period
• Content moderation: Remove inappropriate or spam content
• Database maintenance: Clean up test data or obsolete records

⚠️ Critical Warning: Delete Operations

deleteMany({}) will delete ALL documents in a collection.
db.collection.drop() will delete the entire collection including indexes.
db.dropDatabase() will delete the entire database.

Always test your delete filters with find() first!
Consider using a "soft delete" pattern (adding an "isDeleted" field) for critical data.

🔤 Sorting, Limiting & Skipping — Complete Guide

Query Modifiers Flow

1
Start with find() and optional filter
2
Apply .sort() to order results
3
Use .skip() to paginate through results
4
Apply .limit() to restrict result count
5
Chain with .pretty() for formatted output

Example 1: Sort by single field

db.users.find().sort({ age: 1 })
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e2"), "name": "Young User", "age": 18 }, { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e1"), "name": "Alice Johnson", "age": 29 }, { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e3"), "name": "Senior User", "age": 65 } ]

💡 1 = ascending order, -1 = descending order

Example 2: Sort by multiple fields

db.products.find().sort({ category: 1, price: -1 })
[ { "_id": "PROD003", "name": "Premium Laptop", "category": "Electronics", "price": 1999 }, { "_id": "PROD001", "name": "Basic Laptop", "category": "Electronics", "price": 899 }, { "_id": "PROD002", "name": "Designer Chair", "category": "Furniture", "price": 499 } ]

💡 Sort by category ascending, then by price descending within each category

Example 3: Limit result count

db.users.find().sort({ createdAt: -1 }).limit(5)
[ { /* 5 most recent users */ } ]

💡 .limit(n) restricts the number of documents returned

Example 4: Skip for pagination

db.products.find()
  .sort({ name: 1 })
  .skip(10)
  .limit(10)
[ { /* products 11-20 */ } ]

💡 .skip(n) skips the first n documents. Used with .limit() for pagination

Example 5: Sort with text score

db.products.find(
  { $text: { $search: "laptop" } },
  { score: { $meta: "textScore" } }
).sort({ score: { $meta: "textScore" } })
[ { "_id": "PROD003", "name": "Gaming Laptop Pro", "description": "High-end gaming laptop", "score": 2.5 }, { "_id": "PROD001", "name": "Basic Laptop", "description": "Entry-level laptop for students", "score": 1.8 } ]

💡 Sort by text search relevance score

Example 6: Count documents

db.users.countDocuments({ active: true })
1247

💡 countDocuments() returns the number of documents matching the filter

🚀 Real-World Sorting & Pagination Use Cases

• Product listings: Sort by price, rating, or popularity with pagination
• User management: Display users sorted by registration date or activity
• Content feeds: Show latest posts or most popular content first
• Admin dashboards: Paginate through large datasets for better performance
• Search results: Order by relevance, price, or other criteria

📋 Query Modifiers Cheat Sheet

.sort({field: 1}) - Sort ascending
.sort({field: -1}) - Sort descending
.limit(n) - Limit to n documents
.skip(n) - Skip first n documents
.pretty() - Format output nicely
.countDocuments() - Count matching documents
.explain() - Show query execution plan
.toArray() - Return array (in drivers)

👁️ Projection — Include/Exclude Fields

What is Projection?

Projection allows you to specify which fields to include or exclude in the query results. This improves performance by reducing the amount of data transferred.

// Syntax
db.collection.find(query, projection)

• 1 = include field
• 0 = exclude field
• Cannot mix inclusion and exclusion (except for _id)

Example 1: Include specific fields

db.users.find(
  { age: { $gte: 18 } },
  { name: 1, email: 1 }
)
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e1"), "name": "Alice Johnson", "email": "alice@example.com" }, { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e2"), "name": "Bob Smith", "email": "bob@example.com" } ]

💡 Only name and email fields are returned (plus _id by default)

Example 2: Exclude specific fields

db.users.find(
  {},
  { password: 0, creditCard: 0 }
)
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e1"), "name": "Alice Johnson", "email": "alice@example.com", "age": 29 } ]

💡 All fields except password and creditCard are returned

Example 3: Exclude _id field

db.products.find(
  {},
  { name: 1, price: 1, _id: 0 }
)
[ { "name": "Wireless Mouse", "price": 29.99 }, { "name": "Mechanical Keyboard", "price": 129.99 } ]

💡 _id field can be excluded even when including other fields

Example 4: Project with computed fields (Aggregation)

db.orders.aggregate([
  {
    $project: {
      customerName: "$customer.name",
      total: 1,
      itemCount: { $size: "$items" },
      discountedTotal: { $multiply: ["$total", 0.9] }
    }
  }
])
[ { "customerName": "Alice Johnson", "total": 1099.98, "itemCount": 2, "discountedTotal": 989.982 } ]

💡 Aggregation $project stage allows for complex field transformations

🚀 Real-World Projection Use Cases

• API responses: Return only necessary fields to reduce payload size
• Security: Exclude sensitive fields like passwords, tokens, or personal data
• Performance: Reduce network transfer and memory usage for large documents
• UI optimization: Fetch only fields needed for specific views or components
• Analytics: Project calculated fields for reporting and visualization

📋 Array Operations — Complete Guide

📋 Array Operators Cheat Sheet

$push - Add element to array
$push with $each - Add multiple elements
$pop - Remove first/last element
$pull - Remove elements by condition
$pullAll - Remove all instances of values
$addToSet - Add if not already present
$slice - Limit array size
$position - Specify position for $push

Example 1: $push to add element

db.users.updateOne(
  { name: "Alice Johnson" },
  { $push: { hobbies: "photography" } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $push adds an element to an array field. Creates the array if it doesn't exist.

Example 2: $push with $each for multiple elements

db.users.updateOne(
  { name: "Alice Johnson" },
  { $push: { hobbies: { $each: ["hiking", "cooking"] } } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $each modifier allows adding multiple elements with $push

Example 3: $addToSet to avoid duplicates

db.users.updateOne(
  { name: "Alice Johnson" },
  { $addToSet: { tags: "premium" } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $addToSet adds value only if it doesn't already exist in the array

Example 4: $pull to remove elements

db.users.updateOne(
  { name: "Alice Johnson" },
  { $pull: { hobbies: "photography" } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $pull removes all instances of a value from an array

Example 5: $pull with conditions

db.products.updateOne(
  { _id: "PROD001" },
  { $pull: { reviews: { rating: { $lt: 3 } } } }
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $pull can remove array elements that match a condition

Example 6: $pop to remove first/last element

db.users.updateOne(
  { name: "Alice Johnson" },
  { $pop: { hobbies: -1 } } // -1 = first, 1 = last
})
{ "acknowledged": true, "matchedCount": 1, "modifiedCount": 1 }

💡 $pop removes the first (-1) or last (1) element from an array

Example 7: Querying array elements

db.users.find({ hobbies: "photography" })
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e1"), "name": "Alice Johnson", "hobbies": ["reading", "photography", "hiking"] } ]

💡 Query matches documents where array contains the specified value

🚀 Real-World Array Operations Use Cases

• User interests: Add/remove hobbies, skills, or preferences
• E-commerce: Manage product tags, categories, or review lists
• Social media: Handle followers, likes, or comment threads
• Content management: Store article tags, authors, or related content
• Analytics: Track user activity history or event logs

🔗 $lookup (Join) — Complete Guide

What is $lookup?

$lookup is MongoDB's equivalent of a JOIN in SQL. It performs a left outer join between two collections, adding documents from the "joined" collection to the input documents.

// Basic syntax
db.orders.aggregate([
  {
    $lookup: {
      from: "collection_to_join",
      localField: "field_from_input_docs",
      foreignField: "field_from_joined_docs",
      as: "output_array_field"
    }
  }
])

Example 1: Basic $lookup (orders with customer info)

db.orders.aggregate([
  {
    $lookup: {
      from: "users",
      localField: "customerId",
      foreignField: "_id",
      as: "customerInfo"
    }
  }
])
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e2"), "customerId": "64a1b2c3d4e5f6a7b8c9d0e1", "total": 1099.98, "customerInfo": [ { "_id": "64a1b2c3d4e5f6a7b8c9d0e1", "name": "Alice Johnson", "email": "alice@example.com" } ] } ]

💡 Basic left outer join - adds customer information to each order

Example 2: $lookup with $unwind (flatten results)

db.orders.aggregate([
  {
    $lookup: {
      from: "users",
      localField: "customerId",
      foreignField: "_id",
      as: "customerInfo"
    }
  },
  { $unwind: "$customerInfo" }
])
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e2"), "customerId": "64a1b2c3d4e5f6a7b8c9d0e1", "total": 1099.98, "customerInfo": { "_id": "64a1b2c3d4e5f6a7b8c9d0e1", "name": "Alice Johnson", "email": "alice@example.com" } } ]

💡 $unwind deconstructs the array created by $lookup into individual documents

Example 3: Advanced $lookup with pipeline

db.orders.aggregate([
  {
    $lookup: {
      from: "users",
      let: { customerId: "$customerId" },
      pipeline: [
        { $match: { $expr: { $eq: ["$_id", "$$customerId"] } } },
        { $project: { name: 1, email: 1 } }
      ],
      as: "customerInfo"
    }
  }
])
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e2"), "customerId": "64a1b2c3d4e5f6a7b8c9d0e1", "total": 1099.98, "customerInfo": [ { "_id": "64a1b2c3d4e5f6a7b8c9d0e1", "name": "Alice Johnson", "email": "alice@example.com" } ] } ]

💡 Pipeline syntax allows for more complex joins with additional filtering and projection

Example 4: Multiple $lookup stages

db.orders.aggregate([
  {
    $lookup: {
      from: "users",
      localField: "customerId",
      foreignField: "_id",
      as: "customerInfo"
    }
  },
  { $unwind: "$customerInfo" },
  {
    $lookup: {
      from: "products",
      localField: "items.productId",
      foreignField: "_id",
      as: "productDetails"
    }
  }
])
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e2"), "customerId": "64a1b2c3d4e5f6a7b8c9d0e1", "items": [ { "productId": "PROD001", "quantity": 1 }, { "productId": "PROD002", "quantity": 2 } ], "customerInfo": { /* customer data */ }, "productDetails": [ /* product data */ ] } ]

💡 Multiple $lookup stages can join with different collections in the same pipeline

🚀 Real-World $lookup Use Cases

• E-commerce: Join orders with customer and product information
• Social media: Display posts with user profile data and comment counts
• Content management: Show articles with author information and category details
• Analytics: Combine data from multiple collections for reporting
• User dashboards: Aggregate user data from various collections for display

💡 Performance Tip: $lookup

$lookup can be expensive on large datasets. Consider:
• Creating indexes on the joined fields
• Using $match early in the pipeline to reduce documents
• Denormalizing data if joins are frequent
• Using application-level joins for complex scenarios

📝 Text Search & Regular Expressions

Text Search Setup

Before using text search, you need to create a text index on the fields you want to search:

// Create text index on specific fields
db.products.createIndex({ name: "text", description: "text" })

// Create text index on all string fields
db.products.createIndex({ "$**": "text" })

Example 1: Basic text search

db.products.find({
  $text: { $search: "laptop" }
})
[ { "_id": "PROD001", "name": "Gaming Laptop", "description": "High-performance laptop for gaming and content creation" }, { "_id": "PROD002", "name": "Laptop Stand", "description": "Ergonomic stand for your laptop" } ]

💡 Returns documents containing the search term in any of the text-indexed fields

Example 2: Text search with phrase

db.products.find({
  $text: { $search: "\"gaming laptop\"" }
})
[ { "_id": "PROD001", "name": "Gaming Laptop", "description": "High-performance laptop for gaming and content creation" } ]

💡 Quotes search for exact phrase rather than individual words

Example 3: Text search with exclusion

db.products.find({
  $text: { $search: "laptop -stand" }
})
[ { "_id": "PROD001", "name": "Gaming Laptop", "description": "High-performance laptop for gaming and content creation" } ]

💡 Minus sign excludes documents containing the specified term

Example 4: Sort by text search relevance

db.products.find(
  { $text: { $search: "laptop" } },
  { score: { $meta: "textScore" } }
).sort({ score: { $meta: "textScore" } })
[ { "_id": "PROD001", "name": "Gaming Laptop", "description": "High-performance laptop for gaming", "score": 2.0 }, { "_id": "PROD002", "name": "Laptop Stand", "description": "Ergonomic stand for your laptop", "score": 1.5 } ]

💡 Sort by text search score to show most relevant results first

Example 5: Regular expression search

db.users.find({ name: /^A/ }) // Starts with A
db.users.find({ email: /@gmail\.com$/ }) // Ends with @gmail.com
db.users.find({ name: /son$/i }) // Ends with "son", case-insensitive
[ { "_id": ObjectId("64a1b2c3d4e5f6a7b8c9d0e1"), "name": "Alice Johnson", "email": "alice@gmail.com" } ]

💡 Regular expressions provide powerful pattern matching capabilities

🚀 Real-World Text Search Use Cases

• Product search: Allow users to search product catalog by name, description, or tags
• Content discovery: Enable searching through articles, blog posts, or documentation
• User search: Find users by name, email, or other text fields
• Data analysis: Identify patterns or trends in text data
• Form validation: Use regex to validate email, phone numbers, or other input formats

⚠️ Text Search Limitations

• Text indexes can be large and impact write performance
• Text search doesn't support wildcard searches at the beginning of terms
• Stop words (like "the", "and", "or") are ignored by default
• Case insensitive by default, but diacritic sensitive
• Consider using specialized search engines (Elasticsearch, Algolia) for advanced search needs
